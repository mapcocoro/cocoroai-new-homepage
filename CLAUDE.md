# ココロAI コーポレートサイト

- HTML5 + Tailwind CSS (CDN)

## 開発時の注意

- GitHub Pagesでホスティングしているため、リポジトリはPUBLICのままにすること
- PRIVATEにするとサイトが停止する
- 全ページ共通でヘッダー・フッターを手動で同期する必要あり

## クライアント案件：LINE公式/LINEミニアプリの運用ポリシー（重要）

### バッジの種類（2種類あり別物）

| 種類 | 説明 |
|------|------|
| **LINE公式アカウントの認証バッジ**（灰/青/緑） | 公式アカウントの審査。未認証（灰）でも運用可能 |
| **LINEミニアプリの認証バッジ** | ミニアプリ（LIFF）側の審査。未認証ミニアプリとして公開可能 |

### 目的
ITリテラシーが低いクライアントでも運用でき、かつ「所有権トラブル」や「後から移管できず詰む」を防ぐ。

### 絶対ルール（案件共通）
- **LINE Developers（プロバイダー/チャネル）は必ずクライアント名義で作成**（当社名義では作らない）
- **認証申請はしない（標準スコープ外）**
  - LINE公式アカウントの認証申請（バッジ取得）は対応しない
  - LINEミニアプリの「認証済ミニアプリ」審査申請も対応しない（未認証運用で提供）
- **パスワード預かりっぱなし禁止**
  - 原則：画面共有でクライアントがログイン/認証だけ実施、当社が操作誘導
  - 例外：当社が仮PWで作成した場合は、同日中に「クライアントがPW変更・2FA設定」まで行い引き継ぐ

### 役割分担（ざっくり）
- クライアント：メール/SMSの受信、最終的なアカウント所有、PW変更/2FA
- 当社：制作（Web/LIFF）、LINE Developers設定、導線（リッチメニュー等）設定、動作確認、引き継ぎ資料作成

### 構築フロー（認証申請なしで公開）
1. クライアントのLINE Business IDでログイン（画面共有で誘導）
2. クライアント名のプロバイダー作成（LINE Developers）
3. プロバイダー配下で「LINEミニアプリ」チャネル作成
4. LIFF ID発行・Webアプリ（HTTPS）に設定
5. LIFF URLを導線に設置（公式アカ：リッチメニュー/あいさつ/QRなど）
6. 実機テスト（LINEアプリ内で起動確認）

### 納品時に渡す情報（テンプレ）
- LINE Business ID（メールアドレス）
- LINE Developers：プロバイダー名／チャネル名／チャネルID
- LIFF：LIFF ID／LIFF URL／エンドポイントURL
- Web：公開URL／ホスティング（Vercel等）／ドメイン関連（触った場合）
- 規約/PPのURL
- 緊急時の復旧方法（PWリセット手順など）

### スコープ注記（料金ページとの整合）
- 外部サービス利用料・各種アカウント契約は別途（利用状況により変動）
- 追加の機能開発・大幅デザイン変更は別途見積
